## 一、产品概述

### 1.1 产品定位

**AiVolo.studio** 是一款面向欧洲、南美、东南亚用户的 AI 图片/视频生成工具，主打短视频创作者和普通消费者，通过预制提示词模板降低使用门槛。

### 1.2 核心价值主张

- **简单**：秒懂怎么操作，不需要专业提示词知识
- **酷炫**：生成的内容能上 TikTok 热门
- **便宜**：比外包制作成本低 100 倍

### 1.3 目标市场优先级

1. 欧洲（西班牙/葡萄牙/法国/意大利/德国等）
2. 南美（巴西/墨西哥）
3. 东南亚（印尼/越南/泰国）
4. 北美（美国/加拿大）

### 1.4 目标用户

- 短视频创作者（TikTok/YouTube Shorts/Instagram Reels）
- 受热点影响的普通消费者（被 viral 内容吸引，冲动尝试）
- 电商卖家（第二期迭代方向）

### 4.3 多模型架构

**模型选择原则**：
- 优先选择有高搜索流量的热门模型
- 通过 API 聚合供应商接入
- 约10个模型，全部接入

**模型切换方式**：
- 网页打开时默认选定一个模型
- 用户可在下拉框中选择
- 每个模型显示对应 Logo

**高可用路由层**：
- 主动健康检查：几小时一次
- 被动检测：任务失败一次即切换备用供应商
- 用户无感知（只有 loading 状态）
- 日志记录切换事件

### 4.4 视频生成管线

**异步管线（轮询）**：
- 用户提交请求 → 扣除积分 → 创建任务

- 前端每 2 秒轮询任务状态
- 完成后显示结果

### 5.3 智能缓存层

**缓存匹配逻辑**：
- 使用 `final_prompt`（加工后的提示词）计算相似度
- 相似度 ≥ 85% → 返回缓存结果
- 存储：Postgres `generations_cache` 表 + Redis 热点缓存

**缓存存储**：
| 字段 | 说明 |
|------|------|
| cache_key | 唯一索引 |
| template_id | 模板ID |
| final_prompt_hash | 提示词哈希 |
| scene_dimensions | 场景维度组合 |
| aspect | 比例 |
| result_url | 结果URL |
| result_type | 图片/视频 |
| prompt_embedding | 用于相似度计算 |
| cached_at | 缓存时间 |
| expires_at | 3个月后自动清理 |

**拟态加载**：
- 缓存命中时模拟真实生成流程
- 图片：loading 30秒
- 视频：loading 60秒
- 动画样式与真实API调用完全一致

## 八、API 架构

### 8.1 Provider Adapter Pattern

```
用户层 → 产品封装层 → 模型能力层 → 高可用路由层 → 供应商API
```

**高可用路由层职责**：
- 路由选择
- 故障转移（一次失败即切备用）
- 健康检查（主动 + 被动）

### 8.2 生成管线

**图片生成（同步）**：
```
用户提交 → Postgres RPC（扣积分+创建任务）→ Prompt处理 → 缓存匹配 → API调用 → 返回结果
```

**视频生成（异步）**：
```
用户提交 → Postgres RPC（扣积分+创建任务）→ Prompt处理 → 缓存匹配 → API调用
  ↓
上游 webhook → 更新任务状态
  ↓
前端每2秒轮询 → 显示结果
```
