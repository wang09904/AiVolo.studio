## 一、项目概述

### 1.1 项目背景

**AiVolo.studio** 是一款面向欧洲、南美、东南亚市场的 AI 图片/视频生成工具，主打短视频创作者和普通消费者。核心功能包括文生图、图生图、文生视频、图生视频，通过预制模板系统降低使用门槛。

**核心价值主张**：
- **简单**：秒懂操作，无需专业提示词知识
- **酷炫**：生成内容能上 TikTok 热门
- **便宜**：比外包制作成本低 100 倍

**目标市场优先级**：
1. 欧洲（西班牙/葡萄牙/法国/意大利/德国等）
2. 南美（巴西/墨西哥）
3. 东南亚（印尼/越南/泰国）
4. 北美（美国/加拿大）

### 1.2 开发周期

**21天开发周期**，分3个阶段：

| 阶段 | 天数 | 重点 | 目标 |
|------|------|------|------|
| **第一阶段** | 1-5 | 骨架搭建 | MVP可演示，网站看起来正常，可申请payment accounts |
| **第二阶段** | 6-15 | 主体功能开发 | 生成功能完成，模板系统上线，多模型接入 |
| **第三阶段** | 16-21 | 模板补齐+完善 | 模板功能补齐，进一步完善和发展 |

### 4.3 详细任务分解

#### 板块A：生成功能（executor）

| 功能 | API路由 | 积分消耗 |
|------|---------|----------|
| 文生图 | `/api/generate/image` | 1-3积分/张 |
| 图生图 | `/api/generate/image` | 2-5积分/张 |
| 文生视频 | `/api/generate/video` | 10积分/秒 |
| 图生视频 | `/api/generate/video` | 15积分/秒 |

**生成管线**：Webhook主推 + 前端轮询兜底（视频异步）

#### 板块B：API接入（executor）

| 任务 | 实现 |
|------|------|
| 供应商选择 | API聚合供应商（piapi.ai / grsai.com / pic2api.com） |
| Provider Adapter | 适配器模式，新增模型不改业务代码 |
| 高可用路由层 | 主备切换、健康检查、故障转移 |

**Provider Adapter 接口**：
```typescript
interface ProviderAdapter {
  readonly name: string;
  readonly supportedModels: string[];

  generateImage(params: ImageGenerationParams): Promise<GenerationResult>;
  generateVideo(params: VideoGenerationParams): Promise<GenerationResult>;
  getTaskStatus(taskId: string): Promise<TaskStatus>;
  healthCheck(): Promise<boolean>;
}
```

**接入模型（第一期）**：至少5-10个

| 模型 | 公司 | 类型 | 积分消耗 | 热度 |
|------|------|------|----------|------|
| GPT Image 2 | OpenAI | 图片生成 | 3积分/张 | ⭐⭐⭐⭐⭐ |
| DALL-E 3 | OpenAI | 图片生成 | 3积分/张 | ⭐⭐⭐⭐⭐ |
| GPT Video | OpenAI | 视频生成 | 10积分/秒 | ⭐⭐⭐⭐⭐ |
| Veo 2 | Google | 视频生成 | 10积分/秒 | ⭐⭐⭐⭐⭐ |
| Imagen 3 | Google | 图片生成 | 1积分/张 | ⭐⭐⭐⭐ |
| Stable Diffusion | Stability AI | 图片生成 | 2积分/张 | ⭐⭐⭐⭐ |
| Midjourney | Midjourney | 图片生成 | 3积分/张 | ⭐⭐⭐⭐⭐ |
| Ideogram | Ideogram | 图片生成 | 2积分/张 | ⭐⭐⭐⭐ |
| Flux | Black Forest | 图片生成 | 3积分/张 | ⭐⭐⭐⭐ |
| Luma | Luma AI | 视频生成 | 10积分/秒 | ⭐⭐⭐⭐ |

#### 板块C：积分系统（executor）

| 功能 | 实现 |
|------|------|
| 免费额度 | 新用户赠送20积分 |
| **积分扣除** | **必须用Postgres RPC `create_generation_atomic`原子操作，无超扣风险** |
| 余额展示 | 实时显示积分余额 |
| 积分不足提示 | 引导订阅 |

#### 板块D：模板系统（executor）

| 类别 | 数量 | 说明 |
|------|------|------|
| 热门模板 | 10个 | AI猫猫图、AI换装图、AI梗图等 |
| 电商素材 | 5个 | 产品白底图、场景图等 |
| 室内设计 | 5个 | 现代简约效果图等 |

**模板命名规范**：
- 图片模板：`{类型}_{序号}`，如 `hot_img_01`、`ecommerce_img_01`
- 视频模板：`{类型}_{序号}`，如 `hot_video_01`、`ecommerce_video_01`

#### 板块E：智能缓存层（executor）

| 功能 | 实现 |
|------|------|
| 缓存匹配 | 使用final_prompt计算相似度，阈值85% |
| 缓存存储 | generations_cache表，保存3个月 |
| **拟态加载** | **缓存命中时模拟真实生成流程（图片30秒，视频60秒）** |

**缓存key格式**：
```
cache_key = template_id + final_prompt_hash + aspect + scene_dimensions
```

#### 板块F：用户系统完善（executor）

| 功能 | 实现 |
|------|------|

| 历史记录 | 查看生成历史、可重新下载 |
| 用户素材管理 | 查看+下载自己上传的素材/生成的图片和视频 |

#### 板块G：模型广场（designer + executor）

// src/app/api/webhook/creem/route.ts
export async function POST(request: Request) {
  const body = await request.json();
  const { event_type, data } = body;

  switch (event_type) {
    case 'subscription.created':
      await supabase.from('subscriptions').upsert({
        user_id: data.customer_id,
        plan: data.plan_id,
        status: 'active',
        current_period_end: data.current_period_end
      });
      break;
    case 'subscription.canceled':
      await supabase.from('subscriptions').update({
        status: 'canceled',
        canceled_at: new Date().toISOString()
      }).eq('user_id', data.customer_id);
      break;
  }

  return NextResponse.json({ received: true });
}

## 十、关键设计决策

### 10.1 模型能力层与供应商正交

用户选模型（如GPT image2），高可用路由层选择供应商。用户不感知供应商切换。

### 10.2 同一时刻只有一家供应商服务

故障时自动切换，不影响用户体验。

### 10.3 缓存匹配使用 final_prompt

而非用户原始输入。加工后的prompt更标准化、结构化，匹配更准确。

### 10.4 拟态加载机制

缓存命中时，模拟真实生成流程（图片30秒，视频60秒），确保用户体验一致性。

### 10.5 积分扣除必须用Postgres RPC原子操作

使用 `create_generation_atomic` RPC，确保扣积分和创建任务记录在同一个事务中，无超扣风险。

### 10.6 生成管线：Webhook主推 + 前端轮询兜底

视频生成采用异步管线：Webhook主推回调 + 前端每2秒轮询任务状态作为兜底。

### 10.7 Trust区域：初期不建议假评价

先写自己优势，不做虚假评价。

## 十一、API成本记录机制

### 11.1 成本计算方式

采用时间段订阅模式，不追踪单次API token消耗。成本计算简化如下：

**成本计算公式：**
```
已消耗成本 = 生成次数 × 对应单价（积分定价表）
```

### 11.2 数据记录流程

```
生成任务完成
    ↓
record_generation() RPC（独立事务）
    ├── 记录 generation 元数据（时间、模型、状态）
    └── 关联 subscription 使用额度
```

**generations 表字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| created_at | TIMESTAMP | 生成时间 |
| model_id | VARCHAR(50) | 使用的模型 |
| generation_type | VARCHAR(20) | text_to_image / image_to_image / text_to_video / image_to_video |
| status | VARCHAR(20) | pending / completed / failed |

### 11.3 退款成本计算逻辑

**退款金额计算：**
```
退款金额 = 订单金额 - (已生成次数 × 对应单价)
```

直接使用积分定价表计算，无需记录实际 API 消耗。

**积分定价表（用于退款计算）：**

| 消耗类型 | 单次单价 | 说明 |
|----------|----------|------|
| 文生图（基础模型） | $0.020/张 | Imagen 3 |
| 文生图（高级模型） | $0.050/张 | GPT Image 2 |
| 图生图（基础模型） | $0.030/张 | Imagen 3 |
| 图生图（高级模型） | $0.080/张 | GPT Image 2 |
| 文生视频（标准） | $0.200/秒 | GPT Video |
| 图生视频（标准） | $0.300/秒 | GPT Video |

**退款流程：**
1. 用户申请退款
2. 查询 generations 表，COUNT(*) 计算已生成次数
3. 从订单金额中扣除已生成次数 × 对应单价
4. 剩余金额走 Creem.io 原路退回
5. 扣除已消耗积分对应的 Credits
