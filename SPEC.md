## 一、产品概述

### 1.1 产品定位

**AiVolo.studio** 是一款面向欧洲、南美、东南亚用户的 AI 图片/视频生成工具，主打短视频创作者和普通消费者，提供热门模板、电商素材、室内设计三类模板，通过预制提示词和模板降低使用门槛。

### 1.2 核心价值主张

- **简单**：秒懂怎么操作，不需要专业提示词知识
- **酷炫**：生成的内容能上 TikTok 热门
- **便宜**：比外包制作成本低 100 倍

### 1.3 目标用户

- 短视频创作者（TikTok/YouTube Shorts/Instagram Reels）
- 受热点影响的普通消费者（被 viral 内容吸引，冲动尝试）
- 电商卖家（第二期迭代方向）

### 1.4 目标市场优先级

1. 欧洲（西班牙/葡萄牙/法国/意大利/德国等）
2. 南美（巴西/墨西哥）
3. 东南亚（印尼/越南/泰国）
4. 北美（美国/加拿大）

### 1.5 竞品分析

| 竞品           | 定位      | 优点          | 缺点        | 我们的差异化    |
| ------------ | ------- | ----------- | --------- | --------- |
| **Pollo.ai** | 综合AI视频站 | 模型全、功能多、品牌强 | 价格偏高、界面复杂 | 更简单的模板化体验 |
| **Runway**   | 专业创作者   | 质量高、工具丰富    | 价格贵、学习曲线陡 | 更低门槛、更便宜  |
| **Pika**     | 短视频创作者  | 操作简单、社交传播   | 模型有限      | 模板更多、更易上手 |
| **Kling**    | 快手旗下    | 视频质量好       | 国内限制、需翻墙  | 面向海外、支付便捷 |

### 1.6 竞争优势

1. **模板降低门槛**：预制热门模板，用户无需专业提示词知识
2. **多语言本地化**：优先支持欧洲小语种和南美市场
3. **价格竞争力**：比竞品低 30-50%，适合个人创作者
4. **模型聚合**：一次订阅用所有主流模型

### 2.2 接入模型（MVP 必须）

**模型选择原则**：

- 优先选择有高搜索流量的热门模型（用户会主动搜索这些模型名）
- 通过 API 聚合供应商接入，降低接入成本
- 用 1-3 家多媒体大模型聚合供应商做高可用兜底

**高可用路由设计**：

```
┌─────────────────────────────────────────┐
│          模型能力层（用户看到的）          │
│    GPT image2 / DALL-E 3 / Imagen 3 ...  │
└────────────────┬────────────────────────┘
                 ↓ 用户选择模型
┌─────────────────────────────────────────┐
│        高可用路由层（3家供应商）           │
│                                         │
│   供应商A ←→ 供应商B ←→ 供应商C          │
│   (备用)       (主)       (备用)        │
│                                         │
│   同一时刻只有一家在服务                  │
│   故障时自动切换，不影响用户               │
└─────────────────────────────────────────┘
```

### 2.3 模板系统（第二期扩展，非 MVP 必做）

**设计原则**：

- 文生图 + 图生图共用提示词模板（图生图额外配置参考风格）
- 文生视频 + 图生视频共用提示词模板（图生视频额外配置参考风格）
- 每个核心模板可复用于 2 种生成模式
- **批量生成是一套参数机制**，不是模板分类

**模板数量**：60 个核心模板 = 3 类 × 10 个 × 2（图片/视频各一套）

| 类别       | 文生图/图生图共用的图片模板（10个）                  | 文生视频/图生视频共用的视频模板（10个）      |
| -------- | ------------------------------------ | -------------------------- |
| **热门模板** | AI 猫猫图、AI 换装图、AI 梗图、AI 头像图、AI 节日图... | AI 猫猫视频、AI 换装视频、AI 舞蹈视频... |
| **电商素材** | 产品白底图、场景图、AI 模特图、广告 Banner...        | 产品展示视频、AI 模特视频、广告视频...     |
| **室内设计** | 现代简约效果图、北欧风格图、新中式效果图、软装搭配...         | 装修流程视频、设计方案展示视频...         |

**模板命名规范**：

- 图片模板：`{类型}_{序号}`，如 `hot_img_01`、`ecommerce_img_01`、`interior_img_01`
- 视频模板：`{类型}_{序号}`，如 `hot_video_01`、`ecommerce_video_01`、`interior_video_01`

**批量生成参数**（用户设置，非模板分类）：

- 生成数量：1-5张/个
- 一致性模式：
  - **保持一致**：系列模式，多张图保持视觉一致性（series_generator逻辑）
  - **多样化**：批量模式，多张图在不同场景/角度/光线变化（batch_variation逻辑）

**MVP 阶段**：不实现模板系统，用户直接输入提示词进行生成。模板系统在 MVP 之后作为第二期扩展实现。

**第二期扩展**：

- 室内设计模板扩展（更多风格场景）
- 视频模板扩展

### 3.2 模型广场（SEO 流量策略）

**为什么做模型广场**：

- 竞品分析发现：**AI 视频站 90% 的流量关键词都是模型相关词**
- 每个模型有独立页面 = 每个页面都能接住搜索流量（"Sora 视频生成"、"Kling 2.5 教程"等）
- 新模型发布 = 新流量热点（如 Seedance 出新模型，接下来 1 个月流量狂飙）

**页面结构**：

| 页面    | URL             | 作用                     |
| ----- | --------------- | ---------------------- |
| 模型列表页 | `/models`       | 展示所有可用模型/logo/简介       |
| 模型详情页 | `/model/[slug]` | 每个模型的独立 SEO 落地页 + 生成入口 |

**模型详情页内容**：

- 模型名称 + Logo + 简介
- 该模型支持的生成能力（文生图/图生图/文生视频/图生视频）
- 示例作品展示
- "使用此模型生成"按钮 → 跳转 /create 并预选该模型

**MVP 阶段**：

- 先接入 3-5 个主流模型，每个模型做独立页面
- 后续根据流量数据优先扩展有流量的模型

## 四、用户流程

### 4.1 流程 A：首页直接生成

```
1. 用户打开首页
2. 选择生成类型（文生图/图生图/文生视频/图生视频）
3. 输入提示词 + 上传参考图（可选）
4. 设置参数（模型/比例/分辨率等）
5. 点击"生成" → 跳转到 /create
6. 右侧预览区显示 loading 状态（转圈/闪烁/进度百分比）
7. 生成完成 → 显示结果
8. 下载或分享
```

### 4.2 流程 B：从模板进入

```
1. 用户从首页侧边栏进入模板列表页
2. 浏览模板缩略图/短视频
3. 点击模板缩略图/短视频进入模板详情页
4. 点击"使用此模板"，跳转到 /create，预填充模板参数
5. 用户上传自己的内容（照片/视频） + 输入可变参数
6. 点击"生成"
7. 右侧预览区显示 loading状态（转圈/闪烁/进度百分比）
8. 生成完成 → 显示结果
9. 下载或分享
```

### 4.3 流程 C：从首页推荐模板进入

```
1. 用户从首页推荐模板点击进入模板列表页
2. 浏览模板缩略图/短视频动态图
3. 点击模板缩略图/短视频动态图进入模板详情页
4. 点击"使用此模板"，跳转到 /create，预填充模板参数
5. 用户上传自己的内容（照片/视频） + 输入可变参数
6. 点击"生成"
7. 右侧预览区显示 loading状态（转圈/闪烁/进度百分比）
8. 生成完成 → 显示结果
9. 下载或分享
```

### 4.4 流程 D：注册/订阅

```
1. 用户点击"免费开始"或某功能
2. 弹出登录/注册框（Google OAuth 或邮箱）
3. 注册后获得免费积分
4. 用户消耗积分使用功能
5. 积分不足时引导订阅
```

## 五、设计风格

### 5.1 设计哲学

**高端但有人味，专业但不难懂。**

- 这是一个娱乐创作工具，不是企业级 SaaS
- 各方面风格不要太严肃，要有创意感和活力
- 但也不能太 low，要有品质感

### 5.2 色彩系统

#### 主色（Primary）

```css
:root {
  /* 深紫蓝作为主色，有科技感但不俗气 */
  --color-primary: #7C5CFF;
  --color-primary-hover: #6B4AE8;
  --color-primary-active: #5A3ED4;
  --color-primary-disabled: #B8AFFF;

  /* OKLCH 版本（更现代） */
  --color-primary: oklch(72% 0.18 270);
  --color-primary-hover: oklch(68% 0.18 270);
  --color-primary-active: oklch(64% 0.18 270);
}
```

#### 辅色（Secondary / Accent）

```css
:root {
  /* 珊瑚粉作为点缀色，增加温度感 */
  --color-accent: #FF6B6B;
  --color-accent-hover: #FF5252;

  /* 青绿色作为成功/正向反馈 */
  --color-success: #10B981;
  --color-warning: #F59E0B;
  --color-error: #EF4444;
}
```

#### 中性色（暗色模式）

```css
:root {
  --color-bg: oklch(14% 0 0);           /* 背景：深灰黑 */
  --color-surface: oklch(18% 0 0);      /* 表面：卡片等 */
  --color-surface-elevated: oklch(22% 0 0); /* 悬浮态 */
```

### 6.1 智能缓存层（SmartCache）

**设计思想**：用户完成生成后，将高质量结果存入缓存；当后续用户请求相似时，直接返回缓存，无需调用API。

**缓存匹配逻辑**：

1. 用户输入 → Prompt工程师Agent加工 → 生成结构化的 `final_prompt`
2. **使用 `final_prompt`（加工后的提示词）进行缓存匹配**，而非用户原始输入
3. 原因：加工后的prompt更标准化、结构化，匹配更准确

**缓存命中判断**：

```
缓存key = template_id + final_prompt_hash + aspect + scene_dimensions

相似度计算：
- 计算当前请求的 final_prompt embedding 与缓存中 stored_prompt embedding 的 cosine 相似度
- 相似度 ≥ 85% → 返回缓存结果
- 相似度 < 85% → 调用API生成
```

**缓存价值判断**（非所有生成都值得缓存）：
| 任务类型 | 缓存价值 | 原因 |
|---------|---------|------|
| 模板 + 通用提示词 | 高 | 通用性强，他人可能复用 |
| 模板 + 个性化输入 | 中 | 有一定通用性 |
| 图生图（用户上传参考图） | 低 | 参考图是用户私有，难以匹配 |
| 极具体的个性化描述 | 低 | "XX小区3号楼客厅"，极难复用 |

**缓存存储方案**：

```
├── Supabase Postgres（主存储）
│   └── generations_cache 表
│       ├── cache_key（唯一索引）
│       ├── template_id
│       ├── final_prompt_hash
│       ├── scene_dimensions（场景维度组合）
│       ├── aspect
│       ├── result_url
│       ├── result_type（图片/视频）
│       ├── prompt_embedding（用于相似度计算）
│       ├── cached_at
│       └── expires_at（3个月后自动清理）
│
└── Redis（热点缓存层，LRS淘汰）
```

**缓存策略**：

- 保存时间：3个月
- 命中阈值：相似度 ≥ 85%
- 用户无感知：缓存命中/未命中体验相同，积分相同
- **拟态加载机制**：缓存命中时，不立即显示结果；模拟真实生成流程，显示相同的 loading 状态（30秒-60秒，按任务类型区分），确保用户体验一致性
  - 图片任务：loading 持续 30 秒
  - 视频任务：loading 持续 60 秒
  - loading 期间的动画样式与真实API调用时完全一致
  - 防止用户感知到"瞬间完成"而产生怀疑

### 6.3 生成管线设计

**图片生成管线**（同步，实时返回）：

```
用户提交请求
    ↓
后端：Postgres RPC create_generation_atomic（原子操作：扣积分 + 创建任务记录）
    ↓

    ↓
命中≥85%？→ 是：返回缓存（拟态loading 随机10～30秒，模拟真实图片生成）
    ↓ 否：调用模型能力层
    ↓
高可用路由层：选择可用供应商 → 调用供应商API
    ↓
返回结果给用户 → 更新任务状态为"已完成"
    ↓
若生成失败：自动退还已扣除的积分（通过事务回滚）
```

**视频生成管线**（异步，Webhook主推 + 前端轮询兜底）：

```
用户点生成
    ↓
后端：Postgres RPC create_generation_atomic（原子操作：扣积分 + 创建任务记录）
    ↓
Prompt工程师Agent → 设计编译器 → 缓存层匹配（final_prompt相似度计算）
    ↓
命中≥85%？→ 是：返回缓存（拟态loading 随机30～60秒，模拟真实视频生成）
    ↓ 否：继续
    ↓
调上游 API（带 webhook）
    ↓
上游完成 → webhook 回调 → 更新任务状态
    ↓
前端：每 2 秒轮询任务状态（兜底） / 或 SSE 实时推送
    ↓
前端：显示结果
    ↓
若生成失败：自动退还已扣除的积分（通过事务回滚或独立退款事务）
```

**关键设计**：

- 图片和视频生成均支持智能缓存匹配
- 模型能力层与供应商正交：用户选模型（如GPT image2），高可用路由层选择供应商

## 6.7 API 成本记录机制

### 6.7.1 成本计算方式

由于采用时间段订阅模式，不追踪单次 API token 消耗。成本计算简化如下：

**成本计算公式：**
```
已消耗成本 = 生成次数 × 对应单价（积分定价表）
```

**供应商标注：**
| 供应商 | 状态 |
|--------|------|
| piapi.ai | 待定 |
| grsai.com | 待定 |
| pic2api.com | 待定 |

### 6.7.2 数据记录流程

```
生成任务完成
    ↓
record_generation() RPC（独立事务）
    ├── 记录 generation 元数据（时间、模型、状态）
    └── 关联 subscription 使用额度
```

**generations 表字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| created_at | TIMESTAMP | 生成时间 |
| model_id | VARCHAR(50) | 使用的模型 |
| generation_type | VARCHAR(20) | text_to_image / image_to_image / text_to_video / image_to_video |
| status | VARCHAR(20) | pending / completed / failed |

### 6.7.3 退款成本计算逻辑

**退款金额计算：**
```
退款金额 = 订单金额 - (已生成次数 × 对应单价)
```

直接使用积分定价表计算，无需记录实际 API 消耗。

**积分定价表（用于退款计算）：**

| 消耗类型 | 单次单价 | 说明 |
|----------|----------|------|
| 文生图（基础模型） | $0.020/张 | Imagen 3 |
| 文生图（高级模型） | $0.050/张 | GPT Image 2 |
| 图生图（基础模型） | $0.030/张 | Imagen 3 |
| 图生图（高级模型） | $0.080/张 | GPT Image 2 |
| 文生视频（标准） | $0.200/秒 | GPT Video |
| 图生视频（标准） | $0.300/秒 | GPT Video |

**退款流程：**
1. 用户申请退款
2. 查询 generations 表，COUNT(*) 计算已生成次数
3. 从订单金额中扣除已生成次数 × 对应单价
4. 剩余金额走 Creem.io 原路退回
5. 扣除已消耗积分对应的 Credits

### 7.4 核心卖点（必须突出展示）

**一次订阅 = 所有模型随便用**

这是聚合站最硬的差异化卖点。在 AI 视频市场，通常一个模型（Runway、Pika、Kling、Luma……）要单独开一个订阅。我们主打：**一次订阅，这些模型你都能用**。替用户省了"反复订阅和切换产品"的麻烦。

**展示位置**：

- 首页第二屏（紧跟生成框）
- 定价页卡片显眼位置
- 订阅弹窗左侧

| 套餐          | 原价（月均）  | 折后价（月均）   | 实际年付总额 | 积分/月 | 亮点标签      |

| **Lite 月付** | $15     | $15       | -      | 300  | -         |
| **Lite 年付** | ~~$15~~ | **$10**   | $120/年 | 300  | ⭐ **最划算** |

### 7.6 定价策略核心

- **Lite 年付标注"最划算"**：月均 $10 是最低价，引导用户一次付清年付
- **Pro 年付标注"省 50%"**：对比 Pro 月付 $29，年付 $14.5 省了 50%，引导用户升级到 Pro 版
- **免费版**：让用户试用，降低转化门槛
